概要

  1. String 属于引用类型，不属于基本类型
  2. 用双引号括起来的字符串，例如“abs”,”def”都是存储在“方法区”的“字符串常量池”中

String类中的方法

charAt()

compareTo()

按照字典序比较大小
相等返回0
小返回负数
大返回正数

contains()

判断是否包含字符串

endsWith()

判断当前字符串是否以某字符串结尾

equals()和compareTo()

在Java8中equals()代码中不包含compareTo()…

equalsIgnoreCase()

就是忽略大小写的equals()

getBytes()

就是将字符串转换为byte数组

indexOf()

返回字符串第一次出现起始位置的下标

isEmpty()

判断是否为空串
不能 String s = null 会空指针异常

lastIndexOf()

返回字符串最后一次出现的索引

replace()

将“ab”->“xx”

split()

拆分…

startsWith()

判断是否为某字符串开头

substring()

截取子字符串

toCharArray()

转化为char数组

toLowerCase() 与 toUpperCase()

大小写转化

trim()

取出字符串前后空白

valueOf()

String类方法中 只有一个 valueOf 方法是静态的 不需要 new 对象

StringBuffer

使用“+”进行字符串拼接，如String s = “a”+“b”，会在方法去产生两个字符串对象，在堆内存中产生一个String s对象，频繁使用导致内存空间的浪费

StringBuilder和StringBuffer的区别

StringBuffer中的方法都有：synchronized 关键词修饰，表示StringBuffer在多线程环境下时安全的
StringBuilder是非线程安全的
